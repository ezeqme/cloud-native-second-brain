## Resumo

Logging refers to recording events generated by our applications or systems, and its purpose is to provide us with an understanding of what's happening within the software. When an event is generated, either structured or unstructured, it's delivered to a central place for analysis, usually to the file system or third-party service over the network by using some transport layer that encapsulates the message into Syslog, Forward, OpenTelemetry, or another format. If these events come from different sources and in different formats, at scale present complex challenges, so how can we deal with this heterogeneous nature and make sure we can have a suitable logging layer in our infrastructure?

## Descrição oficial

Logging refers to recording events generated by our applications or systems, and its purpose is to provide us with an understanding of what's happening within the software. When an event is generated, either structured or unstructured, it's delivered to a central place for analysis, usually to the file system or third-party service over the network by using some transport layer that encapsulates the message into Syslog, Forward, OpenTelemetry, or another format. If these events come from different sources and in different formats, at scale present complex challenges, so how can we deal with this heterogeneous nature and make sure we can have a suitable logging layer in our infrastructure? In this presentation, we will do a deep dive into how application logging works around formats, transport, and aggregation. We will share best practices to deal with multiline messages, redaction, and implement a scalable logging layer by using industry standards with Fluent Bit and OpenTelemetry.
