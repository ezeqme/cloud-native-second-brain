# Security Policy

## Reporting a vulnerability

If you find a security issue in this repository, please do **not** open a public issue with sensitive details.

Preferred options:

1. Use GitHub private vulnerability reporting, if enabled for this repository.
2. Otherwise, contact the maintainer privately and include only the minimum information needed to reproduce or assess the issue.

Please include:

- affected file or script;
- impact;
- reproduction steps, when safe;
- suggested fix, if available.

## Sensitive data policy

This repository is intended to be public. Do not commit:

- API keys, tokens, passwords or private keys;
- cloud credentials;
- kubeconfigs or local cluster credentials;
- private `.env` files;
- raw private datasets;
- unpublished third-party content without permission.

Public conference metadata, public URLs and generated notes are acceptable when they respect the source terms.

## Third-party content

This repository references and organizes public conference content. Third-party videos, slides, transcripts, trademarks and event materials remain subject to their original owners' rights and terms.
